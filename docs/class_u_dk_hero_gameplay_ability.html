Detailed Description

Provides hero-specific utility functions. Most hero abilities are Blueprint subclasses of this class.

Complicated abilities (e.g., target locking) can subclass this class and be implemented in C++.

float InBaseDamage,
FGameplayTag InAttackTag,
int32 CurrentComboCount )

Creates a gameplay effect spec handle for hero damage effects.

The returned spec handle can be either:

  • Applied on the enemy character to deal damage, or
  • Applied on the hero character to heal withered health (no need CurrentComboCount in this case)
Remarks
As for the latter case, as the params' names suggest, the use of this function is not quite semantically appropriate. It is kept this way for code reuse purposes (as both cases are involved in player character dealing damage to enemies), but should it be?
