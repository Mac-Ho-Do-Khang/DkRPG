Detailed Description

AI controller that uses Detour Crowd for navigation and avoidance behavior.

Note
  • In order for the algorithm to function properly, some configuration is required.
  • From C++, some parameters are in the category Detour Crowd Avoidance Config of this class.
  • Other parameters are set in the project settings under Project Settings > Engine > Crowd Manager, described below.
Avoidance Config (Index 0..3)
  • Array of avoidance quality presets corresponding to quality levels Low, Medium, Good, and High.
  • Velocity bias:
    • 0: Fully used velocity computed by graph following algorithm
      • No crowd avoidance applied
      • Results in agents sliding as they are trying to reach their targets
    • 1: Fully used velocity computed by Detour crowd avoidance
      • No sliding, but agents may not reach their targets
    • Set to: 0
Sampling Patterns
  • Defines velocity sampling distributions used for avoidance.
  • More patterns improve accuracy but increase CPU cost.
  • Leave empty as default.
Max Agents
  • Maximum number of agents managed by the Detour Crowd system.
  • Agents exceeding this limit will not receive crowd avoidance.
  • Set to: 50
Max Agent Radius
  • Maximum supported agent collision radius (Unreal units).
  • Must be greater than or equal to the largest agent capsule radius.
  • Set to: 300.0
Max Avoided Agents
  • Maximum number of neighboring agents considered for avoidance.
  • Higher values improve behavior in dense crowds at higher CPU cost.
  • Set to: 6
Max Avoided Walls
  • Maximum number of nearby navmesh walls considered for avoidance.
  • Improves navigation around corners and narrow passages.
  • Set to: 8
Navmesh Check Interval
  • Time interval (seconds) between navmesh validity checks.
  • Lower values improve robustness but increase CPU usage.
  • Set to: 1.0
Path Optimization Interval
  • Time interval (seconds) between path smoothing updates.
  • Lower values result in smoother movement and faster correction.
  • Set to: 0.5
Separation Dir Clamp
  • Limits the strength of lateral separation steering.
  • Prevents excessive side movement and jitter.
  • Values:
    • -1: No clamping (full separation force)
  • Set to: 0
Path Offset Radius Multiplier
  • Multiplier applied to path center offset.
  • Controls how far agents deviate from the navmesh centerline.
  • Recommended:
    • 1.0 (default)
    • >1.0 to encourage spreading
    • <1.0 to keep agents centered
  • Set to: 0
Resolve Collisions
  • Enabled: Collision handled by CharacterMovementComponent.
  • Disabled: Collision handled by GraphFollowingComponent.
  • Set to: Enabled
