Detailed Description

Base class for startup data assets, shared by both hero and enemies.

Note
Loading Methods
  • Synchronous Loading:
    • Pros: Returns loaded object from the loading function
    • Cons: Blocks the game thread while loading
    • Apply this for our main Hero character to ensure all data is ready at game start
  • Asynchronous Loading:
    • Pros: Does not block the game thread
    • Cons: Loaded assets appear suddenly
    • Apply this for enemies as there are many of them and they can appear some frames after level start

Gameplay Effects that are applied when the character is initialized.

Used for setting up base attributes, persistent buffs, or static gameplay tags.

Warning
The order of application matters, as some may depend on others being already applied; e.g., GE_DkHero_Startup, which initializes Hero attributes, must be applied before GE_DkHero_Static, which adjusts CurrentHealth based on MaxHealth attribute.
Note
In order to correctly display enemies' attribute set from the editor's debug command showdebug abilitysystem during gameplay, (cycling through enemies with the PGUP and PGDN keys), add this to DefaultGame.ini:
[/Script/GameplayAbilities.AbilitySystemGlobals]
bUseDebugTargetFromHud = true
